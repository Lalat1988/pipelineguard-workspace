Topic 4 — FastAPI Basics
Theory to understand:

What FastAPI is — modern Python web framework for building REST APIs
Why FastAPI — fast, uses async natively, auto-generates API docs, uses decorators for routes
Core concepts: routes, path parameters, query parameters, request body, response model
Pydantic — FastAPI uses it for data validation, define request/response shapes as classes
@app.get(), @app.post() — route decorators you now understand from Topic 2
Automatic docs at /docs — Swagger UI generated automatically

Questions to answer yourself:

What is the difference between path parameter and query parameter?
What does Pydantic do and why is it useful?
Why does FastAPI use async def for route handlers?
